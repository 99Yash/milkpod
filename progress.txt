# Multimodal Video Progress

## Phase 1 - Multilingual Transcript Reliability ✅ (2026-03-09)
Files changed:
- packages/api/src/modules/ingest/model.ts — Added TranscriptionStrategy type ('audio-first' | 'auto' | 'captions-first'), updated ingest body schema
- packages/api/src/modules/ingest/youtube.ts — Added resolveYouTubeAudioUrl() using Innertube streamingData.adaptiveFormats, refactored player API into shared fetchInnertubePlayer()
- packages/api/src/modules/ingest/pipeline.ts — Rewrote orchestratePipeline to support audio-first strategy with captions fallback, extracted transcribeViaAudio/transcribeViaCaptions helpers, passes providerMetadata (transcriptionMethod, fallbackReason)
- packages/api/src/modules/ingest/service.ts — Added optional providerMetadata param to storeTranscript, stored in transcript.providerMetadata jsonb column
- packages/api/src/modules/ingest/index.ts — Route now accepts optional transcriptionStrategy, defaults to 'audio-first'
Outcome: YouTube ingest now defaults to audio-first transcription (ElevenLabs scribe_v2 via audio stream URL), falling back to captions. Transcript metadata records method used and fallback reason. No schema migration needed (providerMetadata jsonb column already existed).

## Phase 2 - Language-Aware Retrieval Heuristic ✅ (2026-03-09)
Files changed:
- packages/api/src/modules/assets/search-service.ts — Replaced FTS-first-with-semantic-fallback with language-aware heuristic: English transcript + Latin query → hybrid ranked search (0.65 lexical + 0.35 semantic normalized scores); non-English → semantic-first with high-confidence lexical supplementation. Added countUnicodeLetters/countLatinLetters helpers.
- packages/api/src/modules/assets/service.ts — Added getTranscriptLanguage(assetId) helper to fetch transcript language for an asset.
- packages/ai/src/system-prompt.ts — Added transcriptLanguage to SystemPromptContext; non-English transcripts get language_guidance section instructing AI to match user's language.
- packages/ai/src/stream.ts — Added transcriptLanguage to ChatRequest interface; threads it through to buildSystemPrompt.
- packages/api/src/modules/chat/index.ts — Fetches transcript language when assetId is present; passes it to createChatStream.
- packages/api/src/modules/shares/index.ts — Fetches transcript language for shared asset chat; passes it to createChatStream.
Outcome: Retrieval now adapts to transcript/query language. English queries on English transcripts use blended lexical+semantic ranking. Non-English (e.g. Hindi) queries use semantic-first search, avoiding English FTS bias. System prompt instructs AI to respond in the transcript's language when non-English. No schema changes needed.

## Phase 3 - Visual Context Extraction and Indexing ✅ (2026-03-09)
Files changed:
- packages/db/src/schema/video-context-segments.ts (new) — New video_context_segment table: id, assetId, startTime, endTime, summary, ocrText, entities (jsonb), confidence, providerMetadata. Indexes on assetId and (assetId, startTime). Cascade delete from media_asset.
- packages/db/src/schema/video-context-embeddings.ts (new) — New video_context_embedding table: id, segmentId (FK to video_context_segment), content, embedding (vector 1536d), model, dimensions. HNSW cosine index for similarity search. Cascade delete from video_context_segment.
- packages/db/src/schemas.ts — Added barrel exports for new video context schemas.
- packages/ai/src/provider.ts — Added visualModel export (Gemini 2.5 Flash) for visual context extraction.
- packages/api/src/modules/ingest/video-context.ts (new) — Visual context extraction module: calls Gemini 2.5 Flash with YouTube URL as file input, extracts timestamped visual segments (20-45 sec windows, max 50 per asset), validates/clamps segments, stores in DB, generates text embeddings from summary+OCR+entities. Uses independent retry with logging-only error handler (doesn't modify asset record). Runs non-blocking after transcript pipeline completes (FR9).
- packages/api/src/modules/ingest/service.ts — Added storeVideoContextSegments() and storeVideoContextEmbeddings() methods to IngestService.
- packages/api/src/modules/ingest/pipeline.ts — Added triggerVisualExtraction() helper that fires extractVideoContext() non-blocking after finalizePipeline completes. Called from all three YouTube transcription paths (captions-first, audio-first, captions fallback). Visual extraction failure does not block transcript readiness.
- packages/db/src/migrations/0018_nostalgic_mikhail_rasputin.sql — Migration creating video_context_segment and video_context_embedding tables with indexes and foreign keys.
Outcome: YouTube video ingest now triggers non-blocking visual context extraction after transcript is ready. Gemini 2.5 Flash analyzes the video via direct URL input, producing timestamped visual segments with summaries, OCR text, and entity lists. Segments are embedded using existing text embedding pipeline and stored with HNSW indexes for similarity search. Visual extraction failures are isolated and do not affect transcript readiness (FR9). Stage-level logging emitted.

## Phase 4 - Ask AI Hybrid Retrieval ✅ (2026-03-09)
Files changed:
- packages/ai/src/retrieval.ts — Added RelevantVisualSegment interface, queryEmbedding option to RetrievalOptions, and findRelevantVisualSegments() function that searches video_context_embedding + video_context_segment tables with cosine similarity. Supports assetId and collectionId scoping. Modified findRelevantSegments() to accept optional pre-computed queryEmbedding to avoid duplicate embedding generation.
- packages/ai/src/types.ts — Added optional visualSegments (RelevantVisualSegment[]) to RetrieveSegmentsOutput. Imported RelevantVisualSegment type.
- packages/ai/src/tools.ts — Extended retrieve_segments tool to search both transcript and visual context embeddings in parallel using a shared pre-computed query embedding. Updated tool description to mention visual context. Tool output now includes visualSegments array and message shows counts for both transcript and visual results.
- packages/ai/src/system-prompt.ts — Updated retrieve_segments tool description to mention visual context retrieval. Added rule 6 instructing AI to distinguish between transcript (what was said) and visual (what was shown) citations.
- packages/db/src/schema/qa.ts — Added qaVisualEvidence table (qa_visual_evidence) with messageId FK to qa_message and videoContextSegmentId FK to video_context_segment. Includes relevance score, unique constraint on (messageId, videoContextSegmentId), and indexes.
- packages/api/src/modules/chat/service.ts — Added extractVisualEvidenceRows() function that extracts visual segment references from retrieve_segments tool outputs. Extended ChatService.saveMessages() to insert visual evidence rows into qa_visual_evidence alongside existing transcript evidence.
- packages/db/src/migrations/0019_stale_winter_soldier.sql — Migration creating qa_visual_evidence table with foreign keys, unique constraint, and indexes.
Outcome: Ask AI retrieval now searches both transcript and visual context embeddings in parallel, returning combined results with separate arrays for transcript segments and visual segments. The AI model receives both modalities and cites visual context (what was shown on screen) alongside transcript content (what was said) in its responses. Visual evidence references are logged to qa_visual_evidence for auditability. Embedding generation is shared across both searches to avoid duplicate API calls. The visualSegments field is optional for backward compatibility with existing frontend code.

## Phase 5 - Hybrid Comments + Comments UI ✅ (2026-03-09)
Files changed:
- packages/db/src/schema/comments.ts (new) — New asset_comment table: id, assetId, userId, body, startTime, endTime, source (comment_source enum: audio | visual | hybrid), evidenceRefs (jsonb for transcript/visual segment ID references), dismissedAt (soft delete). Indexes on (assetId, startTime) and (userId, assetId). Cascade delete from media_asset and user.
- packages/db/src/schemas.ts — Added barrel export for comments schema.
- packages/db/src/migrations/0020_petite_salo.sql — Migration creating comment_source enum and asset_comment table with indexes and foreign keys.
- packages/api/src/types.ts — Added Comment type derived from assetComments schema.
- packages/api/src/modules/comments/model.ts (new) — Elysia type schemas for generate (assetId, regenerate), listQuery (assetId), and feedback (dismiss action) endpoints.
- packages/api/src/modules/comments/service.ts (new) — CommentService abstract static class with list, getById, deleteByAsset, insertMany, and dismissComment methods. Orders comments by startTime ascending.
- packages/api/src/modules/comments/generate.ts (new) — Comment generation pipeline: loads transcript segments and visual context segments, builds 45-second fused evidence windows (transcript + overlapping visual context), sends to Gemini 2.5 Flash Lite for structured comment extraction with source labels, maps comments to DB rows with evidence references, persists via CommentService.insertMany. Max 20 comments per asset, max 30 windows.
- packages/api/src/modules/comments/index.ts (new) — Elysia router at /api/comments with auth macro. POST /generate (creates or returns cached comments, supports regenerate), GET / (list by assetId), POST /:id/feedback (dismiss).
- packages/api/src/index.ts — Mounted comments module.
- apps/web/src/lib/data/queries.ts — Added getComments(assetId, userId) query returning non-dismissed comments ordered by startTime.
- apps/web/src/components/asset/asset-tab-bar.tsx — Added Comments tab (MessageCircle icon, /comments suffix) and updated getActiveTab routing.
- apps/web/src/components/comments/comment-card.tsx (new) — Client component rendering individual comment with timestamp range (clickable for video jump), source badge (Audio/Visual/Hybrid with color coding), comment body, and dismiss action. Uses existing useTimestampAction and VideoMomentDialog for video playback.
- apps/web/src/components/comments/comments-tab.tsx (new) — Client component for comments tab: empty state with generate button, loading state, comment list with regenerate button. Uses Eden treaty client for API calls.
- apps/web/src/app/asset/[id]/comments/page.tsx (new) — Server component page route: authenticates session, loads initial comments via getComments query, renders CommentsTab.
Outcome: Full hybrid comments feature with dedicated data model, generation pipeline, API module, and UI surface. Comments are generated from fused transcript + visual context evidence using Gemini 2.5 Flash Lite. Each comment includes a timestamp range, source label (audio/visual/hybrid), and evidence references for explainability. The UI provides a dedicated Comments tab with timestamp-based video navigation and source-type badges. Comments can be dismissed (soft delete). The generation endpoint caches results and supports regeneration.
