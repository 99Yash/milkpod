## Task 3: Implement dynamic chunk config helper (2000 upper bound)
- **Status**: Complete
- **Files changed**:
  - `packages/api/src/modules/moments/chunking.ts` (new) ‚Äî dynamic chunk config + segment chunker
- **Outcome**: Created `chunking.ts` with three exports:
  1. `getMomentChunkConfig(totalChars)` ‚Äî returns `{ chunkSize, overlap, maxCandidatesPerChunk }` scaling across 5 tiers (900‚Üí2000 chars) per the plan's chunk policy table.
  2. `chunkSegmentsForMoments(segments, config)` ‚Äî groups transcript segments into character-bounded `MomentChunk` windows with overlap, preserving `startTime`/`endTime` and `segmentIds` for traceability. Operates on full segment boundaries (never splits mid-segment), unlike the embedding chunker.
  3. Types: `MomentChunkConfig`, `MomentChunk`.
  Overlap is implemented by rewinding trailing segments into the next chunk. Build and type-check pass across all 7 packages.
